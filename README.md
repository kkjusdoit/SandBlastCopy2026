# Unity 竖屏版《流沙俄罗斯方块》

基于 Unity 6 URP 实现的手机竖屏版《流沙俄罗斯方块》原型项目。

这不是传统“满行消除”的俄罗斯方块，而是结合了：

- Tetromino 下落、移动、旋转
- 锁块后转化为细粒度“沙子”
- 沙粒以元胞自动机规则持续流动
- 仅当同色连通区域从左边界贯通到右边界时触发消除

当前仓库包含一版可运行的 Unity 原型，用于验证核心玩法、沙粒模拟、特殊消除规则和移动端竖屏 UI。

## 项目概览

项目目标是在当前 Unity 空工程中，实现一版手机竖屏、高完成度单机原型，完整还原教程中的核心规则：

- Tetromino 形状生成、移动、旋转、下落
- 锁定后将块体映射为细粒度沙网格
- 沙粒按元胞自动机规则持续流动
- 仅当同色连通区域从左边界贯通到右边界时触发消除

## 实施计划

### Summary

在当前 Unity 6 URP 空项目中实现一版手机竖屏、高完成度单机原型，完整还原教程中的核心规则：

- Tetromino 形状生成、移动、旋转、下落
- 锁定后将块体映射为细粒度沙网格
- 沙粒按元胞自动机规则持续流动
- 仅当同色连通区域从左边界贯通到右边界时触发消除
- 提供完整可玩流程：开始界面、对局、得分、失败、重开、Next 预览、音效与基础反馈

### Key Changes

#### Core Simulation

- 建立双网格系统：
  - 粗网格用于当前活动块的生成、移动、旋转、落地判定
  - 细网格用于沙粒占据、流动与消除检测
- 设定固定换算系数 `grainScale`，每个粗格映射为 `grainScale x grainScale` 个沙粒单元。
- 当前活动块仅在粗网格层存在；落地后一次性写入细网格并转为静态沙粒。
- 碰撞检测按活动块每个粗格映射到细网格覆盖区判断，命中已占据沙粒或越界即视为碰撞。
- 沙粒更新采用自下而上、每行随机列顺序的元胞自动机更新：
  - 下方为空则下落
  - 否则检测左下、右下
  - 双侧都可走时随机选一侧
- 消除检测按颜色分组，从左边界出发做 4 邻域连通搜索；某颜色连通块若接触右边界，则整组进入消除队列。
- 消除流程分为“标记 -> 闪烁/高亮 -> 清空 -> 重力再结算”，允许一次消除后继续触发后续连锁。

#### Game Rules And UX

- 首版保留教程规则，不实现传统整行消除模式。
- 首版使用手机竖屏 + 屏幕按钮输入：
  - 左移
  - 右移
  - 旋转
  - 加速下落
- 提供 `Next` 预览，不做 `Hold`。
- 失败条件定义为：新块生成时即与已有沙粒重叠，无法合法出生。
- 计分先按简洁方案落地：
  - 成功消除按消除沙粒数计分
  - 连续消除可加连击倍率
- 对局节奏分阶段提升：
  - 随时间或分数提高下落速度
  - 仅在不破坏沙粒模拟稳定性的前提下加快局势
- UI 范围限定为高完成度单机原型：
  - 标题/开始
  - 游戏主界面
  - 分数/最高分
  - Next 预览
  - 暂停/失败/重开

#### Rendering And Architecture

- 不使用 Rigidbody/Collider 做沙子，不为每个沙粒创建 GameObject。
- 逻辑层使用纯数据网格驱动：
  - 活动块状态
  - 细网格沙粒颜色/占据状态
  - 消除标记状态
- 表现层采用像素颗粒风渲染，优先用单张可写纹理或等价批量渲染方案，把细网格直接绘制到棋盘区域。
- 活动块单独渲染在沙盘上层，保持“落下前是块、落地后变沙”的视觉分层。
- 模块边界明确拆分为：
  - 游戏状态控制器
  - 活动块规则系统
  - 沙粒模拟系统
  - 连通消除系统
  - 棋盘渲染系统
  - UI/输入系统
- 数据结构预留扩展位，后续可加入：
  - 特殊块
  - 关卡目标
  - 道具
  - 不同消除规则模式

### Test Plan

- 活动块在空盘面中可正常左右移动、旋转、软降和自动下落。
- 活动块贴边、贴已有沙堆时，碰撞检测稳定，不出现穿透或错位。
- 不同形状在锁定时都能正确映射到细网格，不丢格、不越界。
- 沙粒落地后会向下、左下、右下扩散，不会静止成不合理悬空结构。
- 同色区域只在“左边界连到右边界”时消除；未贯通时绝不误消。
- 消除后沙粒继续下落，并允许触发后续二次或多次连锁。
- 新块出生被阻挡时正确结束游戏。
- 竖屏按钮输入在真机分辨率下可稳定操作，UI 不遮挡核心棋盘。
- 在目标移动设备上保持可接受帧率，重点验证高沙粒密度局面。

### Assumptions

- 教程中的核心规则按截图内容完整还原：双网格、沙化、4 邻域、左右贯通消除。
- 第一版目标是“高完成度单机原型”，不包含广告、任务系统、签到、商店、联网或排行榜。
- 美术先以程序化像素颗粒风和简洁 UI 为主，不依赖外部商业资源包。
- 首版只做 `Next`，不做 `Hold`。
- 首版以触屏按钮为主，不同时支持手势输入。

## 运行方式

1. 使用 Unity 6 打开项目。
2. 打开 `Assets/Scenes/SampleScene.unity`。
3. 点击 Play 运行。
4. 运行后会自动生成游戏运行时界面，无需手工搭建场景对象。
